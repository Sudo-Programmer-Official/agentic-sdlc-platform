# Agentic SDLC Platform

This repository is a modular monolith for an AI-driven SDLC orchestration platform.
It provides a FastAPI backend, a Vue 3 frontend, and shared core/agent modules with
clear boundaries for future service extraction.

## Structure
- `apps/web`: Vue 3 + Tailwind + Element Plus frontend
- `apps/api`: FastAPI orchestrator service
- `agent`: Bedrock agent adapters, prompts, and tools
- `core`: SDLC state machine, approvals, ledger, and domain models
- `docs`: product docs and plans
- `infra`: deployment scaffolding

## Requirements Agent Demo
The Requirements Agent is the first autonomous agent implemented in the system. Its purpose
is to capture, structure, and validate requirements at the document layer before any design
or implementation begins.

How It Works
1. A project is created and enters the `REQUIREMENTS_DRAFTED` SDLC stage.
2. A Run is created for this stage.
3. Starting the Run invokes the Requirements Agent.
4. The agent generates structured artifacts:
5. `PRD.md`
6. `USER_STORIES.md`
7. `ACCEPTANCE.md`
8. All actions are logged to the audit ledger.
9. The Run can be paused or canceled at any time.

The agent does not modify code and does not advance SDLC stages automatically. Human approval
is required to proceed.

Example Outputs
Example requirement artifacts generated by the Requirements Agent can be found here:
`docs/examples/requirements`

These examples demonstrate:
- Non-trivial, structured requirements
- Traceable user stories and acceptance criteria
- Early detection of ambiguity and scope risks at the document level

This document-first approach allows requirement issues to be identified and resolved before
design or implementation begins, reducing downstream rework.
