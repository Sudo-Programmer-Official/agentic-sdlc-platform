## Feature Blocks

### 1. Execution Contracts

Define what a run is allowed to touch before AI starts changing code.

- allowed files
- protected zones
- safe refactor zones
- validation steps
- command allowlists
- budget and retry bounds

### 2. Architecture-Aware Runtime

Keep AI work aligned with how your codebase is actually structured.

- package and module awareness
- protected paths
- safe zones
- command coverage
- validation recipes

### 3. Mission Control

Watch every run as it happens.

- current step
- recent commands
- validation state
- artifacts
- failed command visibility
- next best step

### 4. Review Before Trust

Separate planning, implementation, validation, and review so teams can accept AI output without accepting black-box behavior.
